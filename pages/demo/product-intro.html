01 · 项目概述

让 AI 成为电商团队的「智能副驾」，而非「黑盒替代」

多 Agent 电商运营中枢 V3.0 是一套运行于 Windows 环境的大型桌面软件，专为中小型电商团队设计。 系统通过 8 个专业 Agent 协作 + 人工终审 + 本地数据池 + 合规 API 对接， 实现从竞品发现、选品寻源、采购入库到多平台订单履约、资产管理、数据分析的全链路智能化， 彻底告别低效手工操作及定期大规模盘点。

🎯

精准定位

聚焦小红书（营销洞察）+ 美团（订单履约）+ 1688（采购寻源）三平台闭环，不做大而全，做深做透。

🤖

AI 辅助，人工终审

所有涉及资金、库存、采购的关键操作均需人工确认。AI 生成推荐与预填单，支付必须在平台完成，不代付、不托管资金。

🔒

本地化优先

数据库、服务全量本地部署，后端仅监听 127.0.0.1，确保商业数据安全可控，符合数据合规要求。

📊

量化决策看板

借鉴金融量化分析理念，提供商品动量、波动率、毛利估值等指标，支持拖拽式自定义分析画布。

🔴

红色放权机制

极端缺人时可逐级释放 Agent 自主权（L0→L3），全程红色预警、金额硬顶、事后审计，风险可控。

🏭

资产全生命周期

统一管理商超资产（货架、冷柜）和仓库资产（叉车、扫描枪），含保养计划、耗材联动采购。

02 · 痛点分析

中小型电商团队正在面临的六大运营困境

我们深入调研了 20+ 家中小型电商团队的日常运营流程，总结出以下核心痛点，正是本系统要解决的根本问题。

01

竞品信息碎片化，趋势捕捉滞后

运营人员每天手动刷小红书、记录爆款，无法系统化追踪热度变化，错失选品窗口期。平均反应滞后 3-7 天。

02

选品采购依赖个人经验，缺乏数据支撑

1688 上同款商品成千上万，靠肉眼比价、凭感觉下单，毛利率波动大，供应商可靠性无系统评估。

03

多平台订单与库存不同步

美团、自有渠道订单分散管理，库存扣减靠人工，超卖、缺货频发，售后处理效率低。

04

库存管理粗放，盘点耗力

缺乏销量预测和补货建议，要么积压占资金，要么断货丢单。定期全量盘点占用 2-3 人天。

05

资产与耗材管理缺失

货架、冷柜、叉车等固定资产无台账，维修保养靠记忆；包装耗材与商品库存割裂，重复采购或断供。

06

数据报表手工制作，决策缺乏依据

日报、周报靠 Excel 汇总，耗时 1-2 小时/天，数据口径不统一，管理层无法实时掌握经营状况。

04 · 系统架构

四层架构 · 本地优先 · AI 原生设计

系统采用 Electron 桌面壳 + Python FastAPI 本地服务 + 多 Agent 调度引擎的分层架构，所有商业数据本地存储，AI 能力通过 DeepSeek API 按需调用。

总体架构总览

从用户交互层到外部 API 集成的完整技术栈：Electron 多窗口桌面应用 → FastAPI 本地后端（8500 端口，仅 127.0.0.1）→ PostgreSQL / Redis / Qdrant 三存储 → DeepSeek + 1688 + 美团 + 小红书外部 API。

🔴 红色放权机制 — 极端缺人时的安全降级方案

L0 · 完全人工（默认）

任何资金、库存、采购相关操作均需人工确认，Agent 仅做推荐。适合日常人员充足场景。

L1 · 低风险自执行

允许自动发货（库存充足时）、竞品数据自动抓取、补货建议生成。适合晚间 1 人值班。

L2 · 中等自主

高置信度采购单自动生成（金额 < 上限）、售后意图自动分类。适合周末无人值守。

L3 · 高自主（红色警戒）

销量预测自动下单、紧急补货匹配供应商。全程红色横幅、金额硬顶、事后审计报告。

❌ 永远禁止自主执行：实际资金支付、大额退款、隐私数据导出、删除关键日志

🎛️ 指挥官仪表盘

Agent 待办卡片瀑布流，按紧急程度排序，支持 Ctrl+Enter 快捷审批、批量操作、备注记录。

📦 商品管理

Product/SKU CRUD、分类树、搜索筛选、库存历史，支持多规格 SKU 与图像向量关联。

📊 库存看板

库存流水、盘点调整、StockCheck 表单，流水仅 INSERT 不可删除，冲红通过负记录实现。

🛒 订单管理

多平台订单列表、详情 Drawer、发货/退款操作，库存不足自动挂起。

🔍 竞品分析

🛍️ 采购管理

拖拽画布（grid-layout-plus）、12 个内置预设图表、4 数据源 × 11 指标 × 5 维度 DSL 查询、模板保存/加载。

📄 报告中心

Agent 自动生成 HTML 日报列表、iframe 预览、PDF 导出、定时调度配置。

🏭 资产台账

资产树形分类、CRUD、QR 码生成打印、移动记录、保养计划管理。

🔴 红色控制台

红色放权 L0-L3 启用/关闭、授权范围 Checkbox、审计审核表、审计日志。

⚙️ 设置中心（9 标签）

通用/安全/LLM 提供商/Agent 提示词/Agent 运行/平台 API/通知/存储/关于，Agent 提示词 9 项独立可配置。

07 · 安全与合规

五位一体安全体系 · 数据合规 · 操作可溯

本系统不代付、不托管资金，所有外部 API 调用经过统一网关记录日志，支持限流和缓存。支付行为必须在 1688/美团等平台完成，系统仅采集行为数据作为对账依据。

🔐

身份认证

JWT（access 30min + refresh 7d）+ RBAC 四角色（管理员/运营主管/仓管员/财务只读）+ TOTP 二次验证

📝

操作审计

所有敏感操作写入 audit_logs，不可篡改；红色放权期间独立 red_audit 标记，保留 90 天

🛡️

网络安全

后端仅监听 127.0.0.1；Electron 操作超时自动锁屏；API Key AES 加密存储

💾

数据安全

PostgreSQL 本地存储；BitLocker / AES 字段加密；导出文件自动加水印（时间+操作人）

⚠️

异常监控

检查官 Agent 实时 ML 异常检测 + 规则引擎；严重预警推送任务中心要求人工处理

🔏

软件完整性

安装包签名校验；配置文件哈希锁定；非法修改自动锁定并提示
