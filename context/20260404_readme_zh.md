## 项目定位

本仓库为 **HealthJian** 的静态个人博客站点，采用 HTML、CSS、JavaScript 实现，无打包构建，适合部署在 **GitHub Pages** 或任意静态文件服务。站点包含技术文章、项目展示、生活随笔、小说章节、更新日志等模块。

---

## 当前能力摘要

- **主题与语言**：明暗主题切换；全站导航与带 `data-zh` / `data-en` 的文案中英文切换（`language.js`）。Markdown 阅读页支持 `*_zh.md` 与 `*_en.md` 配对，依赖 `sitewide-language-change` 事件重新加载正文。
- **博客总览**：`pages/blog.html` 从 `js/blog.js` 的 `getBlogPosts()` 读取数据，支持分类筛选、搜索与分页；分类键需与页面筛选按钮一致（含 **技术、面试、生活、随想、思想、摘抄、论文** 等）。
- **首页**：`js/latest-articles.js` 展示最新 3 篇文章（同源数据）。
- **Markdown 阅读器**：`pages/blog/moban_new_md.html`，通过 `?md=` 指定 `context/` 下文件；集成目录（含 h2/h3 折叠）、代码高亮、KaTeX、侧栏抽屉、打印/分享/全屏等。
- **更新日志**：`pages/changelog.html`，可与首页动态联动。

文章评论区等为 **前端占位**，未接后端；搜索以客户端过滤为主。

## 技术栈

- HTML5、CSS3（含变量、Flex/Grid）、ES6+ JavaScript  
- Markdown：`marked`、代码高亮 `highlight.js`、公式 `KaTeX`  
- 部分页面使用 **Tailwind CDN**（如博客总览）  
- 图标：**Font Awesome 6**

## 目录结构（开发常用路径）

| 路径 | 说明 |
|------|------|
| `/context/` | Markdown 源稿；推荐中英成对 `xxx_zh.md` + `xxx_en.md` |
| `/js/blog.js` | 博客列表数据源 `getBlogPosts()` |

| `/css/blog-post.css` | 文章页侧栏、目录、抽屉与操作区样式 |
| `/pages/changelog.html` | 站点更新记录 |

更完整的树状说明见仓库根目录 `README.md` 第 4 节。

## 新增博客文章流程

1. 在 `context/` 撰写 Markdown；若需中英切换，使用 **`文件名_zh.md` 与 `文件名_en.md`**，列表中的 `url` 建议指向 `*_zh.md`。  
2. 在 `js/blog.js` 的 `getBlogPosts()` 数组中追加一条记录（`id`、`title` / `title_en`、`excerpt`、`date`、`url`、`category`、`tags`）。  
3. 若在 `pages/blog.html` 增加新分类，需同步增加筛选按钮与 `categoryLabel` 映射。  
4. 本地用 HTTP 服务预览（勿依赖 `file://` 直接打开，以免 `fetch` 失败）。

## Markdown 模板要点

- 访问形式：`/pages/blog/moban_new_md.html?md=../../context/你的文件.md`  
- 单文件 `xxx.md`：英文界面会尝试加载 `xxx_en.md`，不存在则仍读原文件。  
- 渲染与目录逻辑见 `markdown-renderer.js`；样式见 `markdown-styles.css` 与 `blog-post.css`。  
- 模块级说明另见 `pages/blog/blogjs_markdown/README.md`。

## 本地预览与部署

**预览：** 在仓库根目录启动静态服务器，例如 `python -m http.server 8080`，浏览器访问对应端口。

**部署：** GitHub **Settings → Pages** 指定分支与根目录；注意路径大小写与资源引用一致性。
