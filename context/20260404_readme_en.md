## What this project is

A **static personal site** for HealthJian, built with HTML, CSS, and JavaScript—no bundler required. It is a good fit for **GitHub Pages** or any static host. The site includes technical posts, project pages, essays, novel chapters, and a changelog.

---

## Current capabilities

- **Theme & language:** Light/dark mode; global Chinese/English UI via `language.js` (`data-zh` / `data-en`). The Markdown reader pairs `*_zh.md` with `*_en.md` and reloads on the `sitewide-language-change` event.
- **Blog index:** `pages/blog.html` reads `getBlogPosts()` from `js/blog.js`, with category filters, search, and pagination. Category keys must match filter buttons (e.g. **tech, interview, life, thoughts, idea, excerpt, paper**).
- **Home:** `js/latest-articles.js` shows the three newest posts from the same data source.
- **Markdown reader:** `pages/blog/moban_new_md.html` loads files under `context/` via `?md=`; features TOC (including collapsible h2/h3 groups), syntax highlighting, KaTeX, a collapsible sidebar, print/share/fullscreen actions.
- **Changelog:** `pages/changelog.html`, optionally synced with the home “updates” panel.

Comment areas are **UI placeholders** only (no backend). Search is client-side over the blog dataset.

## Adding a blog post

1. Add Markdown under `context/`; for bilingual posts use **`basename_zh.md`** and **`basename_en.md`**, and point the list `url` at the `*_zh.md` file (or symmetrically `_en`).  
2. Append an object to the array returned by `getBlogPosts()` in **`js/blog.js`** (`id`, titles, excerpts, `date`, `url`, `category`, `tags`).  
3. If you add a **new category**, add a filter button and `categoryLabel` entry in **`pages/blog.html`**.  
4. Preview over **HTTP** (e.g. `python -m http.server`); `file://` often breaks `fetch`.

## Markdown template essentials

- Pattern: `/pages/blog/moban_new_md.html?md=../../context/your-file.md`  
- For a lone `file.md`, the English UI tries **`file_en.md`** first, then falls back to the original path.  
- Rendering and TOC logic: **`markdown-renderer.js`**; styles: **`markdown-styles.css`**, **`blog-post.css`**.  
- Module-level notes: **`pages/blog/blogjs_markdown/README.md`**.
