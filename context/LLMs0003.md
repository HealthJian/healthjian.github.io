# 如何查看或确认模型训练完成，提示词工程与模型部署教程

## 1. 如何确认模型已经训练好？

### 1.1 查看训练日志

在训练过程中，通常会生成日志文件，记录训练进度和性能指标。你可以通过以下方式查看模型的训练情况：

```bash
# 检查训练日志
tail -f ./logs/training.log
````

该命令将实时输出日志文件中的内容，你可以查看训练的损失函数（Loss）、准确度（Accuracy）等关键指标。

### 1.2 评估模型性能

训练完成后，我们可以通过一些评估标准来判断模型的性能，例如：

* **BLEU分数**：用于衡量生成文本与真实文本的相似度。
* **F1得分**：用于分类任务中评估模型的准确度和召回率。
* **人工评估**：让团队成员或相关专家对模型输出的简历进行人工评估，给出满意度评分。

可以通过以下方式执行评估：

```bash
# 评估模型
python evaluate_model.py --model ./output_model --test_data ./test_data.csv
```

如果模型能够在这些评估标准下表现出色（如BLEU>42，人工满意度>4.2/5），则说明模型训练完成且表现良好。

### 1.3 通过推理确认

你也可以通过直接推理测试模型，查看其输出结果。以下是如何使用模型进行简历生成的例子：

```python id="fj3hvq"
from transformers import AutoModelForCausalLM, AutoTokenizer

# 加载训练好的模型和分词器
model = AutoModelForCausalLM.from_pretrained("./output_model")
tokenizer = AutoTokenizer.from_pretrained("Qwen/Qwen-7B")

# 测试输入
input_text = "我是一名软件工程师，拥有5年开发经验，熟悉Python和Java。"

# 生成简历内容
inputs = tokenizer(input_text, return_tensors="pt")
output = model.generate(inputs["input_ids"], max_length=150)

# 输出生成的简历
generated_text = tokenizer.decode(output[0], skip_special_tokens=True)
print(generated_text)
```

如果模型能根据输入生成符合预期的简历内容，那么说明模型已经训练好，可以投入使用。

## 2. 提示词工程：如何优化模型生成结果

提示词工程（Prompt Engineering）是通过精心设计输入提示来控制和优化模型输出的过程。在你的课题中，提示词工程将帮助你更好地生成与职位描述匹配的简历。

### 2.1 提示词设计

设计合适的提示词是确保生成内容质量的关键。以下是一些优化提示词的策略：

1. **明确目标职位**：在输入简历时，明确指出目标职位，帮助模型生成更加精准的简历内容。

   ```text
   "请根据以下简历生成一份适合软件开发工程师岗位的简历："
   ```

2. **具体细化要求**：如果希望简历中的某些部分突出，可以在提示词中加入具体要求。

   ```text
   "请突出工作经验和技术技能，并确保简历符合Python开发岗位要求："
   ```

3. **动态Few-Shot学习**：通过提供几个样本简历（Few-Shot）来帮助模型理解如何生成特定类型的简历。

   ```text
   "以下是一个软件工程师简历示例，生成一份类似的简历，突出我的Python技能："
   ```

4. **限制生成长度**：为了避免生成内容过长或过短，指定最大长度。

   ```text
   "请生成一份不超过500字的IT行业简历："
   ```

### 2.2 通过微调提升生成效果

LoRA（Low-Rank Adaptation）技术允许你通过小规模微调来调整模型，使其在特定领域（如简历生成）表现得更好。你可以通过以下命令来微调模型：

```bash
# 使用LoRA进行微调
python finetune_lora.py --model Qwen/Qwen-7B --train_data train_data.csv --output_dir ./output_model --lora_rank 8 --epochs 3
```

此命令将微调Qwen模型，使其更适合你的简历生成任务。

## 3. 将训练好的模型投入使用

### 3.1 部署模型

你可以使用**FastAPI**或**Flask**等框架将训练好的模型部署为Web服务，供用户在线使用。以下是使用FastAPI部署Qwen模型的示例：

```python id="khg9jv"
from fastapi import FastAPI
from transformers import AutoModelForCausalLM, AutoTokenizer
import torch

# 初始化FastAPI
app = FastAPI()

# 加载训练好的模型
model = AutoModelForCausalLM.from_pretrained("./output_model")
tokenizer = AutoTokenizer.from_pretrained("Qwen/Qwen-7B")

@app.post("/generate_resume/")
def generate_resume(input_text: str):
    # 将输入转换为模型可接受的格式
    inputs = tokenizer(input_text, return_tensors="pt")
    
    # 使用模型生成简历
    output = model.generate(inputs["input_ids"], max_length=300)
    
    # 解码并返回生成的简历
    generated_text = tokenizer.decode(output[0], skip_special_tokens=True)
    return {"generated_resume": generated_text}
```

### 3.2 创建API端点

通过以上代码，你创建了一个API端点，当用户发送简历生成请求时，FastAPI会调用训练好的Qwen模型进行推理，生成简历并返回。

使用命令启动FastAPI服务：

```bash
uvicorn app:app --reload
```

启动后，你可以通过以下URL访问生成简历的API：

```
http://127.0.0.1:8000/generate_resume/
```

用户可以通过HTTP请求将输入的职位描述和简历信息传递给API，API会返回生成的简历。

### 3.3 生成简历的客户端请求

你可以通过`curl`或编写前端界面来访问API。以下是使用`curl`进行POST请求的示例：

```bash
curl -X 'POST' \
  'http://127.0.0.1:8000/generate_resume/' \
  -H 'Content-Type: application/json' \
  -d '{
  "input_text": "请根据以下简历生成一份适合软件开发工程师岗位的简历：我是一名软件工程师，拥有5年开发经验，熟悉Python和Java。"
}'
```
